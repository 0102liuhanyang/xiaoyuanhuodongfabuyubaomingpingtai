# 需求分析

## 项目概述
校园活动发布与报名管理平台面向学生、组织者、管理员提供活动发布、报名、审核、签到、统计与通知等能力，满足校内活动的全流程管理。

## 角色与目标
### 学生
- 浏览活动、筛选/排序、查看详情
- 报名/候补/取消报名
- 签到参与、查看我的报名
- 接收并查看通知

### 组织者
- 创建/编辑活动，提交审核
- 管理报名名单、导出 CSV
- 生成签到码，查看签到情况
- 查看单活动统计

### 管理员
- 审核活动（通过/驳回）
- 监管活动（下架/归档/生成签到码）
- 黑名单管理
- 发送系统通知/广播
- 查看汇总统计

## 功能需求
### 登录与权限
- 注册/登录，基于 JWT
- 角色权限控制（学生/组织者/管理员）

### 活动管理
- 活动创建、编辑、提交审核
- 活动状态流转：draft → pending_review → published → canceled/rejected/archived
- 活动列表/详情/筛选/排序

### 报名与签到
- 报名/候补报名、取消报名
- 候补转正（容量释放或扩容时）
- 签到码生成与签到校验

### 通知与统计
- 通知列表、标记已读
- 组织者单活动统计
- 管理员汇总统计

## 非功能需求
- 安全：JWT 鉴权，接口权限校验
- 可用性：统一主题与清晰交互
- 可维护性：模块化前后端分离
- 性能：分页查询，避免全量加载

## 约束与假设
- 数据库使用 MySQL
- 前后端分离，前端通过 `/api` 代理访问后端
- 当前类别、状态规则以代码内配置为准
